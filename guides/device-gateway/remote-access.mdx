Devices deployed behind firewalls and NATs are difficult to reach for troubleshooting and administration.
ngrok solves this by creating outbound tunnels from the device, giving technicians and end users direct access using their existing tools.

## How it works

1. [Install ngrok](https://ngrok.com/download) on a device or gateway in the remote network
2. Start a tunnel for the protocol you need (SSH, RDP, HTTP, or TCP)
3. Share the ngrok URL with the technician or end user
4. They connect using their existing tools (SSH client, RDP client, browser)

## Quick example

<Tabs>
  <Tab title="HTTP">
    Expose a web-based admin dashboard (or any HTTP service):

    ```bash
    ngrok http 8080
    ```
  </Tab>
  <Tab title="SSH">
    Expose [SSH on a remote Linux device](/using-ngrok-with/ssh):

    ```bash
    ngrok tcp 22
    ```
  </Tab>
  <Tab title="RDP">
    Expose [RDP on a remote Windows device](/guides/ssh-rdp/):

    ```bash
    ngrok tcp 3389
    ```
  </Tab>
</Tabs>

See [the device gateway quickstart](/guides/device-gateway/quickstart) for further steps.

## One gateway for an entire network

You don't need an ngrok agent on every device.
You can run a single agent on one gateway machine and route to any device on the local network. The following is an example [ngrok agent config](/agent/config/) that does just that:

```yaml title="ngrok.yml"
version: 3
agent:
  authtoken: YOUR_NGROK_AUTHTOKEN

endpoints:
  - name: plc-ssh
    url: tcp://plc-ssh.internal:22
    upstream:
      url: tcp://192.168.1.50:22
  - name: admin-dashboard
    url: https://dashboard.internal
    upstream:
      url: http://192.168.1.100:3000
```

Each endpoint is addressable inside ngrok's network, and you can connect them to [Cloud Endpoints](/universal-gateway/cloud-endpoints/) for external access. Cloud Endpoints are long-lived endpoints that run in the ngrok cloud service.

## Using the API

You can use the [Agent API](/agent/api/) to start and stop tunnels:

```bash
# Start a tunnel on demand
curl -X POST http://localhost:4040/api/tunnels \
  -H "Content-Type: application/json" \
  -d '{"name":"dashboard","proto":"http","addr":"3001","domain":"dashboard.example.com"}'
```

## Secure access with authentication

Add authentication so only authorized users can reach your devices.
See the [Traffic Policy guide](/guides/device-gateway/traffic-policy) for details on adding OAuth, JWT validation, and IP restrictions.

The following example demonstrates a Traffic Policy that uses basic auth to secure traffic to your endpoints:

<CodeGroup>

```yaml policy.yml
on_http_request:
  # Prompt for username and password with up to 10 credential pairs
  - actions:
      - type: basic-auth
        config:
          credentials:
            - username: user1
              password: password123
            - username: user2
              password: securepass456
```

```json policy.json
{
  "on_http_request": [
    {
      "actions": [
        {
          "type": "basic-auth",
          "config": {
            "credentials": [
              {
                "username": "user1",
                "password": "password123"
              },
              {
                "username": "user2",
                "password": "securepass456"
              }
            ]
          }
        }
      ]
    }
  ]
}
```

</CodeGroup>
