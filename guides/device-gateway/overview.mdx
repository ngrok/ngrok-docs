ngrok makes it easy to [securely connect to devices](https://ngrok.com/use-cases/device-gateway) deployed in remote networks.
Whether you need to reach a sensor behind a firewall, provide remote admin access to a kiosk, or connect your cloud to thousands of field-deployed devices, ngrok gives every device a secure, addressable endpoint.
No firewall changes, VPNs, or open ports required.

## How does it work?

The ngrok agent runs on a device or a gateway machine in the remote network and opens a single outbound TLS connection to the ngrok cloud on port 443.
No inbound ports, firewall rules, or VPN setup required on the remote network.
Your cloud services, technicians, and customers then reach the device through standard URLs that ngrok manages for you.

You don't need an agent on every device.
One agent on a gateway machine can route to any device on the local network behind it—or you can install the agent per-device if your topology requires it.

Any TCP-based protocol works.
HTTP, TLS, SSH, RDP, Modbus, and other proprietary industrial protocols all travel through the same outbound connection.
Your technicians keep using their existing tools—only the gateway machine running the ngrok agent needs to be updated.

The ngrok agent is a lightweight, dependency-free binary available for Linux, Windows, ARM64, Raspberry Pi, Docker, and Kubernetes.
You can also embed connectivity directly in your own application using the [ngrok SDK](/guides/device-gateway/sdk)—useful when your software already runs in your customer's environment.

For a deeper look at tunnels and protocol support, see [Secure Tunnels](/guides/device-gateway/tunnels).
For platform-specific setup, see the [installation guides](/guides/device-gateway/linux).

## Why use ngrok as a device gateway?

- **Every device gets a URL:** Each device becomes addressable by your cloud, technicians, and customers without touching the remote network.
- **Policy lives in the cloud:** Update access controls, authentication, and rate limits without firmware pushes or truck rolls.
- **Scales via API:** Every feature and function is available via API so you can manage your entire fleet programmatically.
- **No VPN required:** Unlike a VPN, the ngrok agent connects outbound on port 443 and scopes access to specific services—not entire networks. No inbound ports, firewall rules, or network reconfiguration needed on the remote network.
- **Pay only for what you use:** On Pay-as-you-go plans, billing is per active endpoint hour. Spin endpoints up and down via API for security reasons, not cost ones.
