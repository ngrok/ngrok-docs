ngrok helps you securely access APIs, databases, and other private services running in remote networks through [site-to-site connectivity](https://ngrok.com/use-cases/site-to-site-connectivity). This is useful for securely connecting to your customers' on-prem environments, branch offices, data centers, and more, all without opening inbound ports or managing VPN infrastructure.

You can run the ngrok agent inside remote networks, then [securely connect from your Kubernetes cluster](/guides/site-to-site-connectivity/customer-networks), enforce access control with IP restrictions, white-label connections with a custom connect URL, run redundant agents for high availability, and more.

## Why use ngrok for site-to-site connectivity?

<CardGroup cols={2}>
  <Card title="No inbound ports" icon="shield">
    The ngrok agent connects outbound over TLS on port 443. No firewall changes or open ports required in the remote network.
  </Card>
  <Card title="No VPN complexity" icon="plug">
    Replace VPN tunnels, VPC peering, and custom networking with a single agent and a Cloud Endpoint.
  </Card>
  <Card title="Centralized traffic control" icon="traffic-light">
