Webhooks require a publicly accessible URL, which makes local development challenging. ngrok solves this by giving your local server a public HTTPS endpoint that webhook providers can deliver events to.

## How it works

1. Start your local webhook handler on any port
2. Run `ngrok http <port>` to create a public URL
3. Register the ngrok URL with your webhook provider
4. Receive webhook events directly on your local machine

## Quick example

```bash
ngrok http 3000
```

Copy the HTTPS forwarding URL from the agent console and paste it into your webhook provider's configuration (e.g., Stripe, GitHub, Twilio).

## Inspect webhook payloads

ngrok's built-in [traffic inspector](/guides/share-localhost/inspection) lets you see every webhook request in real time, including headers, body, and response. This makes it easy to debug webhook integrations without adding logging to your application.

## Replay failed webhooks

Instead of waiting for the webhook provider to retry a failed delivery, you can [replay the request](/guides/share-localhost/replay) instantly from the ngrok traffic inspector. This dramatically speeds up the development cycle when building webhook handlers.

## Verify webhook signatures

Use the [Verify Webhook Origin Traffic Policy action](/traffic-policy/actions/verify-webhook-origin/) to validate that incoming webhooks are authentic before they reach your application.
