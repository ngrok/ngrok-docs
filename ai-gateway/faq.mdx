## Access keys

### What are access keys?

Access keys authenticate requests to `https://gateway.ngrok.ai`. When you use one with built-in OpenAI or Anthropic providers, the AI Gateway can supply upstream credentials—you aren't required to have your own provider account or provider API keys. See [Access keys](/ai-gateway/concepts/api-keys).

---

### How are access keys different from ngrok API keys?

**Access keys** authenticate your app's LLM requests to `gateway.ngrok.ai`. **ngrok API keys** authenticate to the ngrok platform API (`api.ngrok.com`) for managing tunnels, domains, and other ngrok resources. **AI Gateway API keys** authenticate to the AI Gateway API at `api.ngrok.ai` (`/access-keys`, `/provider-keys`, etc.) for automation.

---

### Can I create multiple access keys?

Yes. Create separate keys for each client or application. This enables independent revocation and per-client usage tracking.

---

### What happens if I lose my access key?

The token is only shown once at creation time. If you lose it, delete the old key and create a new one.

---

### Are access keys scoped to endpoints?

No. Access keys are account-scoped and work with `https://gateway.ngrok.ai`. You do not need a custom endpoint URL.

---

### Which providers work with access keys?

OpenAI and Anthropic support ngrok.ai inference without a provider key. For other built-in providers (such as Google or DeepSeek), add a provider key in app.ngrok.ai to bring your own key. See [Attaching provider keys](/ai-gateway/guides/attaching-provider-keys).

## Credits & billing

### Do I need a ngrok subscription plan?

No—not to use the AI Gateway itself. Sign in at [app.ngrok.ai](https://app.ngrok.ai), purchase AI Gateway credits, and send traffic to `https://gateway.ngrok.ai`.

If you want ngrok platform features on the same account—team members, or internal endpoints for [custom providers](/ai-gateway/concepts/custom-providers)—manage those through [dashboard.ngrok.com](https://dashboard.ngrok.com). That typically requires a plan such as [Pay-as-you-go](https://ngrok.com/pricing). See [Credits](/ai-gateway/concepts/credits).

---

### What are AI Gateway credits?

AI Gateway credits are a prepaid balance you purchase in [app.ngrok.ai](https://app.ngrok.ai). They are separate from billing on [dashboard.ngrok.com](https://dashboard.ngrok.com).

Every request consumes credits for the AI Gateway's processing fee. When the AI Gateway supplies the upstream key (OpenAI and Anthropic), credits also cover model cost. With [provider keys](/ai-gateway/guides/attaching-provider-keys), upstream costs go to your provider—credits still cover the processing fee. See [Credits](/ai-gateway/concepts/credits).

---

### What's the minimum credit purchase?

$5.00. Purchase credits from the [Credits page](https://app.ngrok.ai/credits).

---

### Do credits expire?

Yes. Credits expire **365 days** after purchase.

---

### What happens when credits run out?

The gateway rejects new requests until you purchase more credits from the [Credits page](https://app.ngrok.ai/credits).

---

## General

### Does ngrok.ai provide AI provider API keys?

No. ngrok.ai does not provision or hand out provider API keys. However, with an [access key](/ai-gateway/concepts/api-keys) and [credits](/ai-gateway/concepts/credits), you can use certain built-in providers (OpenAI and Anthropic) directly through ngrok.ai without needing your own provider account or provider API keys. 

Those providers' charges are deducted from your AI Gateway credits, on top of the gateway's processing fee. For other providers, you must bring your own  provider API keys.

## Configuration and setup

### Why am I getting "provider not allowed"?

The provider falls outside the access scope of the access key's [configuration](/ai-gateway/guides/access-key-configurations). Add the provider to the configuration's allowed providers, or send the request with an access key whose configuration allows it. See [Error Codes](/ai-gateway/reference/error-codes).

---

### Why am I getting "model unknown"?

Either the model falls outside the configuration's access scope, or it's a non-catalog model sent without a provider prefix. Add the model to your [access key configuration](/ai-gateway/guides/access-key-configurations)'s allowed models, or prefix unknown models with their provider:

```json
{
  "model": "openai:gpt-5-preview",
  "messages": [{"role": "user", "content": "Hello"}]
}
```

---

### How do I limit which models users can access?

Set the access scope on the access key's [configuration](/ai-gateway/guides/access-key-configurations) to allow only the models you want. See [Create a configuration](/ai-gateway/guides/access-key-configurations#create-a-configuration).

---

### Can I use the gateway with the Vercel AI SDK?

Yes:

```typescript
import { createOpenAI } from "@ai-sdk/openai"

const openai = createOpenAI({
  baseURL: "https://gateway.ngrok.ai/v1",
})
```

See [SDK Integration](/ai-gateway/sdks) for details.

## Failover

### How does failover work?

When a request fails, the gateway tries the next candidate in its list. This includes failures from:

- Timeouts
- HTTP errors (4xx, 5xx)
- Connection failures

Failover order:
1. Next API key for the same model/provider
2. Next model candidate (from model selection or client `models` array)

The gateway never retries the same model/key combination—it always moves to the next candidate.

See [Error Handling](/ai-gateway/guides/error-handling) for details.

---

### How long does the gateway try before giving up?

Set the **Total timeout** in [Account Settings](/ai-gateway/guides/account-settings). It caps the time spent across all failover attempts for a request.

---

### Can I disable automatic failover?

Not directly, but you can narrow the candidates: attach a single provider key per provider, have clients request a single model, and set short timeouts in [Account Settings](/ai-gateway/guides/account-settings).

---

### Will the gateway failover to a different provider automatically?

Yes, if the client specifies multiple models that span different providers:

1. Using the `models` array: `"models": ["openai:gpt-4o", "anthropic:claude-3-5-sonnet-20241022"]`
2. Or using just the model name (like `"model": "gpt-4o"`) when selection strategies return candidates from multiple providers

The gateway automatically tries alternative providers when the primary fails. Specify fallback models in the request body:

```json
{
  "model": "gpt-4o",
  "models": ["claude-3-5-sonnet-20241022"],
  "messages": [{"role": "user", "content": "Hello"}]
}
```

## Keys and authentication

### What's the easiest way to authenticate requests?

Send an [access key](/ai-gateway/concepts/api-keys) with every request. The AI Gateway supplies the upstream provider credentials server-side. See [Securing Your Gateway](/ai-gateway/guides/securing-endpoints).

---

### Do I need to configure provider API keys in the gateway?

Every request uses an [access key](/ai-gateway/concepts/api-keys). For built-in OpenAI and Anthropic, ngrok.ai inference works with [credits](/ai-gateway/concepts/credits) and no stored provider keys. For other providers—or to use your own OpenAI/Anthropic accounts—store [provider keys](/ai-gateway/guides/attaching-provider-keys) and attach them in an [access key configuration](/ai-gateway/guides/access-key-configurations).

---

### How do I secure my gateway when using my own provider keys?

Every request to `https://gateway.ngrok.ai` presents a valid access key. The AI Gateway stores provider keys in app.ngrok.ai and injects them server-side, so they stay out of your client code. See [Securing Your Gateway](/ai-gateway/guides/securing-endpoints).

---

### Can I use different keys for different teams?

Yes. Create separate [access keys](/ai-gateway/concepts/api-keys) for each team or client. Assign different [configurations](/ai-gateway/guides/access-key-configurations) to scope providers, models, and credentials per team.

---

### How do I rotate API keys?

**Access keys**: Delete the old key and create a new one. Update your clients.

**Provider keys**: Add the replacement, update the configuration routing rule, verify traffic, then delete the old key. See [Provider Keys](/ai-gateway/guides/attaching-provider-keys#rotating-provider-keys).

---

## Performance and costs

### Does the gateway add latency?

Yes, a small amount for parsing, token counting, and routing—typically ~10-15ms, though it varies with your region and network path to the gateway.

Provider response time dominates total latency.

---

### How are tokens counted?

Using **tiktoken** (OpenAI's tokenizer) for estimation, with provider-reported counts used when available.

Token counts appear in [usage events](/ai-gateway/observability#usage-api).

---

### Does the gateway charge for token usage?

Every request through the gateway consumes [credits](/ai-gateway/concepts/credits). Credits cover the AI Gateway's processing fee on each request. When the AI Gateway supplies the upstream key (OpenAI and Anthropic), credits also cover model cost. With [provider keys](/ai-gateway/guides/attaching-provider-keys), upstream costs go to your provider accounts.

---

### Can I cache responses to reduce costs?

Currently no. Caching may be added in future versions.

---

## Security and privacy

### Is my data stored by ngrok?

The AI Gateway doesn't retain your full request and response bodies—they're processed in memory. It does store some request details for usage events and debugging:

- Request and response bodies are processed in memory, not persisted by default.
- The AI Gateway records request details such as token counts, latencies, and other metadata for usage events and debugging.

---

### Can I use the gateway for sensitive data?

Yes, with a few considerations:

- Review your compliance requirements.
- Remember that requests still reach your upstream AI providers.

---

### How do I redact PII automatically?

Automatic request and response modification isn't available on `https://gateway.ngrok.ai` today. Redact or transform sensitive data in your application before sending it to the gateway.

## Models and providers

### Which providers are supported?

Built-in support for:

- OpenAI
- Anthropic
- Groq
- Google
- DeepSeek
- OpenRouter
- Hyperbolic
- InceptionLabs
- Inference.net

Plus any self-hosted OpenAI or Anthropic Claude API compatible endpoint. See the [Model Catalog](/ai-gateway/reference/model-catalog) for the full list.

---

### Can I use self-hosted models?

Yes. See [Create a custom provider](/ai-gateway/concepts/custom-providers#create-a-custom-provider).

---

### How do I know which models are available?

Built-in providers are listed in the [model catalog](/ai-gateway/reference/model-catalog). Custom providers declare models when you create them.

---

### Can clients use models not in the catalog?

Yes, if the client includes a provider prefix (like `openai:new-model`), the gateway passes the request through when the provider is allowed by the access key's configuration.

To restrict to catalog models only, set an allowlist in the [access key configuration](/ai-gateway/guides/access-key-configurations).

---

### Why isn't my custom model working?

Ensure:

1. The upstream is OpenAI or Anthropic Claude API compatible
2. The custom provider is created with the correct model IDs and base URL
3. The access key's configuration allows the provider and has routing credentials attached
4. The endpoint is reachable from ngrok

## Monitoring and debugging

### How do I view request metrics?

Query usage with the AI Gateway API, or check the **Usage** page in app.ngrok.ai when available. See [Observability](/ai-gateway/observability).

---

### Why are my requests failing silently?

Start with the error code and message in the response body, then review usage in [Observability](/ai-gateway/observability) and confirm your [access key configuration](/ai-gateway/guides/access-key-configurations) allows the provider and model you requested. See [Debugging](/ai-gateway/guides/debugging) and [Error Codes](/ai-gateway/reference/error-codes).

---

### How do I debug which provider was used?

View [usage events](/ai-gateway/observability#usage-api) in [app.ngrok.ai](https://app.ngrok.ai) or query them with the AI Gateway API. Per-provider and retry detail in usage events is on the [roadmap](/ai-gateway/observability#roadmap).

---

### Can I see individual request/response bodies?

The AI Gateway records usage metadata such as token counts—not full request or response bodies. See [Observability](/ai-gateway/observability) for what's captured.

## Advanced usage

### Can I implement custom routing logic?

Use [access key configurations](/ai-gateway/guides/access-key-configurations) to set provider and model allowlists and routing rules per access key. Use the `models` array in requests for failover between models.

See [Model selection](/ai-gateway/guides/model-selection-strategies).

---

### How do I prioritize certain models or providers?

List models in preference order in the request body:

```json
{
  "model": "openai:gpt-4o",
  "models": ["anthropic:claude-sonnet-4-6"],
  "messages": [{"role": "user", "content": "Hello"}]
}
```

Or create separate access keys with different configurations for different clients.

---

### Can I route based on request content?

Not at the gateway layer today. Use separate access keys and configurations per client or route in your application before calling the gateway.

---

### Does the gateway support streaming?

Yes, streaming is fully supported. The gateway forwards SSE streams transparently.

## Troubleshooting

### The gateway is timing out

Increase timeouts in [Account Settings](/ai-gateway/guides/account-settings).

Or check provider health metrics.

---

### I'm getting rate limited even with multiple keys

[Add multiple provider keys](/ai-gateway/guides/attaching-provider-keys#add-a-provider-key) for the provider and attach them in your [access key configuration](/ai-gateway/guides/access-key-configurations). The gateway tries them in order when one fails. See [Key selection and failover](/ai-gateway/guides/key-selection-failover).

---

### Failover isn't working

For **key failover** (same provider, different keys):
- Configure multiple keys for the provider
- Keys are tried in order when one fails

For **provider failover** (different providers):
- Client must specify fallback models using the `models` array
- Or the same model must be available from multiple providers

```json
{
  "model": "gpt-4o",
  "models": ["claude-3-5-sonnet-20241022"],
  "messages": [{"role": "user", "content": "Hello"}]
}
```

For **model selection failover**:
- Configure multiple strategies in `model_selection`
- Strategies are tried in order until one returns models
