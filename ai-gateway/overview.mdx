The AI Gateway routes your app's requests to AI providers behind a single endpoint and [access key](/ai-gateway/concepts/api-keys). OpenAI and Anthropic work without provider accounts—add [provider keys](/ai-gateway/guides/attaching-provider-keys) or [custom providers](/ai-gateway/concepts/custom-providers) to reach other models.

## Why use the AI Gateway?

<CardGroup cols={2}>
  <Card title="Start without provider accounts" icon="key">
    Use built-in OpenAI and Anthropic with an access key and credits—no OpenAI or Anthropic signup required.
  </Card>
  <Card title="Automatic Failover" icon="rotate">
    If one provider or key fails, the gateway automatically tries the next model or key.
  </Card>
  <Card title="Compatible With Popular SDKs" icon="plug">
    Works with official and third-party SDKs. Point the base URL at `gateway.ngrok.ai` and use your access key.
  </Card>
  <Card title="Self-Hosted Models" icon="server">
    Route to local models like Ollama or vLLM alongside cloud providers using [custom providers](/ai-gateway/concepts/custom-providers) and [provider keys](/ai-gateway/guides/attaching-provider-keys).
  </Card>
</CardGroup>

## Quick example

Point your SDK at the gateway and use your [access key](/ai-gateway/concepts/api-keys):

<CodeGroup>
```python Python (OpenAI) highlight={4-5}
from openai import OpenAI

client = OpenAI(
    base_url="https://gateway.ngrok.ai/v1",
    api_key="ng-xxxxx-g1-xxxxx"  # Your access key
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
```

```python Python (Anthropic) highlight={4-5}
import anthropic

client = anthropic.Anthropic(
    base_url="https://gateway.ngrok.ai",
    api_key="ng-xxxxx-g1-xxxxx"  # Your access key
)

message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "Hello!"}]
)
```

```typescript TypeScript (OpenAI) highlight={4-5}
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://gateway.ngrok.ai/v1",
  apiKey: "ng-xxxxx-g1-xxxxx"  // Your access key
});

const response = await client.chat.completions.create({
  model: "gpt-4o",
  messages: [{ role: "user", content: "Hello!" }]
});
```

```typescript TypeScript (Anthropic) highlight={4-5}
import Anthropic from "@anthropic-ai/sdk";

const client = new Anthropic({
  baseURL: "https://gateway.ngrok.ai",
  apiKey: "ng-xxxxx-g1-xxxxx",  // Your access key
});

const message = await client.messages.create({
  model: "claude-opus-4-6",
  max_tokens: 1024,
  messages: [{ role: "user", content: "Hello!" }],
});
```

```bash cURL (OpenAI) highlight={3}
curl https://gateway.ngrok.ai/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer ng-xxxxx-g1-xxxxx" \
  -d '{
    "model": "gpt-4o",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

```bash cURL (Anthropic) highlight={3}
curl https://gateway.ngrok.ai/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: ng-xxxxx-g1-xxxxx" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-opus-4-5",
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```
</CodeGroup>

On each request, the AI Gateway:

1. Receives your request with your access key at `gateway.ngrok.ai`
2. Validates the key and loads your account configuration
3. Selects which model and provider to use
4. Forwards the request using ngrok.ai inference or your provider keys
5. Retries with the next key or model on failure
6. Returns the response

## What can you do?

| Use Case | Description |
|----------|-------------|
| **Use without provider accounts** | Get started with OpenAI and Anthropic using an access key and [credits](/ai-gateway/concepts/credits) |
| **Per-key routing** | [Access key configurations](/ai-gateway/guides/access-key-configurations) scope providers, models, and credentials |
| **Multi-model failover** | List multiple models in a request; the gateway tries each in order |
| **Multi-key failover** | Attach multiple provider keys in a configuration routing rule |
| **Self-hosted models** | Route to Ollama, vLLM, or other servers via [custom providers](/ai-gateway/concepts/custom-providers) |

<Note>
Every request consumes [credits](/ai-gateway/concepts/credits) for the AI Gateway's processing fee. You don't need a specific ngrok subscription plan to get started—purchase credits from the [Credits page](https://app.ngrok.ai/credits). See [Credits](/ai-gateway/concepts/credits) for how billing works and when you might need a ngrok plan for platform features like team members or internal endpoints.
</Note>
