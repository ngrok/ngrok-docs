On `https://gateway.ngrok.ai`, you choose models in each request. [Access key configurations](/ai-gateway/guides/access-key-configurations) restrict which providers and models an access key may call.

## Choosing a model in requests

| Approach | Example | Use when |
|----------|---------|----------|
| Explicit model | `"model": "gpt-4o"` | You know which model to use |
| Provider prefix | `"model": "openai:gpt-4o"` | Force a specific provider |
| Multiple models | `"models": ["gpt-4o-mini", "claude-sonnet-4-6"]` | Fail over if the primary model fails |

See [Models](/ai-gateway/concepts/models) and the [model catalog](/ai-gateway/reference/model-catalog).

### Example

```json
{
  "model": "gpt-4o",
  "models": ["gpt-4o-mini", "anthropic:claude-sonnet-4-6"],
  "messages": [{"role": "user", "content": "Hello"}]
}
```

The gateway tries `gpt-4o` first, then each entry in `models` in order, until one succeeds.

## Restricting providers and models

Use an [access key configuration](/ai-gateway/guides/access-key-configurations) to limit what each access key can reach:

```json
{
  "access": {
    "providers": { "allow": ["openai", "anthropic"] },
    "models": { "allow": ["openai/gpt-4o", "anthropic/claude-sonnet-4-6"] }
  }
}
```

Requests outside the allowlist are rejected before they reach a provider.

## Routing credentials per provider

Within a configuration, **routing rules** decide whether ngrok.ai inference or your [provider keys](/ai-gateway/guides/attaching-provider-keys) authenticate to each provider. See [Access Key Configurations](/ai-gateway/guides/access-key-configurations#routing-rules).
