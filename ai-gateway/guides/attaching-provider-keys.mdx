Provider keys are the API keys you already have with OpenAI, Google, Anthropic, and other providers. You add them to the AI Gateway once, and it uses them to call those providers for you—the keys stay on the gateway instead of in your app.

Your app talks to the gateway with an [access key](/ai-gateway/concepts/api-keys). Which provider key the gateway uses for a given request comes from that access key's [configuration](/ai-gateway/guides/access-key-configurations). See [Access keys vs provider keys](/ai-gateway/concepts/api-keys#access-keys-vs-provider-keys) for how the two differ.

<Note>
For built-in **OpenAI** and **Anthropic**, you can skip provider keys and use [ngrok.ai inference](/ai-gateway/providers/openai) on your [credits](/ai-gateway/concepts/credits) instead.
</Note>

## Add a provider key

### Via dashboard

<Steps>
  <Step title="Open Providers">
    Go to [Providers](https://app.ngrok.ai/providers) and pick the provider from the **Built-in** or **Custom** tab.
  </Step>

  <Step title="Add a key">
    On the provider's detail page, add a key under **API keys**.
  </Step>
</Steps>

### Via the AI Gateway API

You can add your own API Keys for providers via the [Provider Keys API](/api-reference/provider-keys/create):

```bash
curl -X POST https://api.ngrok.ai/provider-keys \
  -H "Authorization: Bearer $AI_GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "providerId": "openai",
    "name": "My OpenAI Key",
    "value": "sk-proj-..."
  }'
```

After you add a key, the dashboard only shows it redacted (`first10***last4`)—keep your own copy from the provider. Provider keys are account-wide, so you can associate one key with many access keys.

<Note>
Adding a key stores it but doesn't route any traffic to it. To make it active, [attach it to an access key configuration](#using-provider-keys).
</Note>

## Using provider keys

A provider key does nothing on its own. The gateway only uses it once it's attached to an access key configuration and that configuration is assigned to an access key. Here's the full flow, from key to working request.

<Steps>
  <Step title="Add the provider key">
    [Add a key](#add-a-provider-key) for the provider if you haven't already.
  </Step>

  <Step title="Attach the key in an access key configuration">
    In an [access key configuration](/ai-gateway/guides/access-key-configurations#create-a-configuration), add a routing rule for the provider, choose **Bring your own API key**, and select your key. Attach more than one key to the same provider and the gateway tries them in the order you list, failing over when one is rate-limited or rejected—see [Key selection and failover](/ai-gateway/guides/key-selection-failover).

    With the AI Gateway API, reference the key's ID in the configuration's router rules:

    ```json
    {
      "router": {
        "rules": [
          {
            "provider": "openai",
            "steps": [
              {
                "type": "user",
                "keySelectionStrategy": "ordered",
                "keys": [{ "id": "aigpk_xxxxx" }]
              }
            ]
          }
        ]
      }
    }
    ```
  </Step>

  <Step title="Assign the configuration to an access key">
    On the [Access Keys](https://app.ngrok.ai/keys) page, assign the configuration to the [access key](/ai-gateway/concepts/api-keys) your app uses.
  </Step>

  <Step title="Send a request">
    Your app sends only the access key. The gateway matches it to the configuration, picks the provider key, and calls the provider:

    ```python
    from openai import OpenAI

    client = OpenAI(
        base_url="https://gateway.ngrok.ai/v1",
        api_key="ng-xxxxx-g1-xxxxx"  # Access key, not the provider key
    )

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{"role": "user", "content": "Hello!"}]
    )
    ```
  </Step>
</Steps>

## Rotating provider keys

[Rotating a key](/api-reference/provider-keys/rotate) updates its value in place and keeps the same key ID, so every access key configuration that points to it keeps working—no configuration changes needed.

<Steps>
  <Step title="Rotate the key">
    Send the new value to the [Provider Keys API](/api-reference/provider-keys/rotate), authorized with your [AI Gateway API key](/ai-gateway/concepts/api-keys#credentials-overview):

    ```bash
    curl -X POST https://api.ngrok.ai/provider-keys/aigpk_xxxxx/rotate \
      -H "Authorization: Bearer $AI_GATEWAY_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "My OpenAI Key",
        "value": "sk-proj-..."
      }'
    ```
  </Step>

  <Step title="Verify traffic">
    Send a test request with your access key. The key ID is unchanged, so there's nothing else to update.
  </Step>
</Steps>
