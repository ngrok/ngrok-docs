AI Gateway API Keys let you authenticate requests to your AI Gateway. When you use one, ngrok handles the provider API keys for you — you don't need accounts with OpenAI, Anthropic, or other providers.

## How they work

1. You [create an AI Gateway API Key](#creating-keys) via the dashboard, CLI, or API
2. Each key is scoped to a specific AI Gateway endpoint (via the required `endpoint_id`)
3. You use the key as the `apiKey` value in your SDK or `Authorization: Bearer` header
4. ngrok validates the key and automatically injects its own provider API keys
5. Your request is forwarded to the provider

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://your-ai-gateway.ngrok.app/v1",
    api_key="ng-xxxxx-g1-xxxxx"  # Your AI Gateway API Key
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)
```

Your AI Gateway API Key token is never sent to the provider — only ngrok's managed provider keys are used.

## Supported providers

Currently, the following providers have ngrok-managed keys:

| Provider | Supported |
|----------|-----------|
| **OpenAI** | ✅ |
| **Anthropic** | ✅ |
| Google | ❌ (use [BYOK](/ai-gateway/bring-your-own-keys/overview)) |
| DeepSeek | ❌ (use [BYOK](/ai-gateway/bring-your-own-keys/overview)) |
| Other providers | ❌ (use [BYOK](/ai-gateway/bring-your-own-keys/overview)) |

This list may grow over time. For providers not listed above, use [Bring Your Own Keys](/ai-gateway/bring-your-own-keys/overview).

## Automatic model selection

When using AI Gateway API Keys, you can set `model: "ngrok/auto"` and the gateway will automatically select from available ngrok-managed providers:

```json
{
  "model": "ngrok/auto",
  "messages": [{"role": "user", "content": "Hello"}]
}
```

No explicit provider configuration is needed — the gateway selects the best available model from OpenAI and Anthropic.

## Key format

Tokens have the format `ng-{id}-g1-{secret}` (for example, `ng-2abc123def-g1-7Hj8Kp3Qr5Tv9Wx`). The token is only shown once at creation time — store it securely.

<Warning>
If you lose your API key token, you cannot retrieve it. Delete the old key and create a new one.
</Warning>

## Creating keys

### Dashboard

1. Navigate to [**AI Gateways**](https://dashboard.ngrok.com/ai-gateways) in the dashboard
2. Select your gateway
3. Go to the **Keys** tab
4. Click **Create Key**
5. Copy the token immediately — it won't be shown again

### CLI

```bash
ngrok api ai-gateway-api-keys create \
  --endpoint-id ep_xxxxx \
  --description "My app"
```

The `--endpoint-id` flag is required. The endpoint must have the AI Gateway action enabled.

### API

```bash
curl -X POST https://api.ngrok.com/ai_gateway_api_keys \
  -H "Authorization: Bearer $NGROK_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Ngrok-Version: 2" \
  -d '{
    "endpoint_id": "ep_xxxxx",
    "description": "My app"
  }'
```

## Managing keys

| Operation | CLI | API |
|-----------|-----|-----|
| List | `ngrok api ai-gateway-api-keys list` | `GET /ai_gateway_api_keys` |
| Get | `ngrok api ai-gateway-api-keys get <id>` | `GET /ai_gateway_api_keys/{id}` |
| Update | `ngrok api ai-gateway-api-keys update <id>` | `PATCH /ai_gateway_api_keys/{id}` |
| Delete | `ngrok api ai-gateway-api-keys delete <id>` | `DELETE /ai_gateway_api_keys/{id}` |

## Key properties

| Property | Description |
|----------|-------------|
| `id` | Unique identifier (for example, `aigk_...`) |
| `uri` | API resource URI |
| `created_at` | Creation timestamp (RFC 3339) |
| `updated_at` | Last update timestamp (RFC 3339) |
| `description` | Human-readable description (max 255 characters) |
| `metadata` | Arbitrary user-defined metadata (max 4096 characters) |
| `created_by` | Reference to the user who created the key |
| `last_updated_by` | Reference to the user who last updated the key |
| `endpoint` | Reference to the associated AI Gateway endpoint |
| `token` | The API key token (only returned on creation) |
| `display_name` | Display-safe name for the key |
| `last_used` | Last usage timestamp (eventually consistent — may lag behind actual usage) |

## Relationship to credits

AI Gateway API Keys require [prepaid credits](/ai-gateway/concepts/credits). When credits run out, all keys stop working. Credits cover both the processing fee and the upstream provider fee when using AI Gateway API Keys.

## Authentication failure behavior

If a key with the `ng-*-g1-*` format is presented but is invalid, the request is **rejected** — it does not fall through to passthrough mode.

| Error code | Description |
|------------|-------------|
| [`ERR_NGROK_27007`](/errors/err_ngrok_27007) | The provided API key is invalid or does not exist |
| [`ERR_NGROK_27008`](/errors/err_ngrok_27008) | This API key is scoped to a different gateway |
| [`ERR_NGROK_27009`](/errors/err_ngrok_27009) | Failed to validate API key (temporary failure) |

## Mixed mode (AI Gateway API Keys + BYOK)

AI Gateway API Keys and BYOK can coexist in the same gateway. When you authenticate with an AI Gateway API Key:

- **Providers with BYOK keys configured**: Your BYOK keys take precedence
- **Providers without BYOK keys**: ngrok-managed keys are injected automatically

For example, if you configure BYOK keys for Google and authenticate with an AI Gateway API Key:
- OpenAI and Anthropic → use ngrok-managed keys
- Google → uses your BYOK keys

This lets you use AI Gateway API Keys for supported providers while bringing your own keys for others.

## Difference from other ngrok credentials

| Credential | Purpose |
|------------|---------|
| **AI Gateway API Keys** | Authenticate requests to AI Gateway endpoints. Used by your application to make AI requests. |
| **ngrok API Keys** | Authenticate to the ngrok management API (`api.ngrok.com`). Used for managing ngrok resources. |
| **ngrok Authtokens** | Authenticate ngrok agents to the ngrok platform. |
| **Provider API Keys** | Keys from OpenAI, Anthropic, etc. Used in [BYOK mode](/ai-gateway/bring-your-own-keys/overview). Not needed when using AI Gateway API Keys for supported providers. |
