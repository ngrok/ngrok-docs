title: Custom Providers
description: Connect custom endpoints and self-hosted models to the AI Gateway.
---

**Custom providers** are upstream endpoints you define, such as self-hosted Ollama or vLLM, a private deployment, or any OpenAI- or Anthropic-compatible API not in the [built-in catalog](/ai-gateway/reference/model-catalog).

See [Create a custom provider](#create-a-custom-provider) below, or the [Providers API](/api-reference/providers/create).

Traffic still authenticates to the gateway with an [access key](/ai-gateway/concepts/api-keys). Attach provider keys and routing rules through an [access key configuration](/ai-gateway/guides/access-key-configurations).

## Requirements

Custom providers must:

1. **Expose a supported API:** OpenAI Chat Completions, OpenAI Responses, or Anthropic Messages format
2. **Be reachable from ngrok:** public HTTPS URL or an ngrok internal endpoint
3. **Declare models:** list callable model IDs in the provider definition

## Create a custom provider

### Via the ngrok dashboard

1. Navigate to [app.ngrok.ai](https://app.ngrok.ai) → **Providers** → **Custom** → **Add provider**.
2. Choose **External URL** or **Local** (Ollama, LM Studio, etc.).
3. Set a provider ID, base URL, API format, and models.
4. Add a provider key if the upstream requires authentication.

### Via the AI Gateway API

Create custom providers with the [Providers API](/api-reference/providers/create). Authorize the request with your [AI Gateway API key](/ai-gateway/concepts/api-keys#credentials-overview).

```bash
curl -X POST https://api.ngrok.ai/providers \
  -H "Authorization: Bearer $AI_GATEWAY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "providerId": "my-ollama",
    "name": "Local Ollama",
    "baseUrl": "https://my-ollama.internal",
    "supportedApiSurfaces": [{ "format": "openai", "surface": "chat-completions" }],
    "models": [{ "modelId": "llama3.2" }]
  }'
```

See the [Providers API reference](/api-reference/providers/create) for all operations.

## URL requirements

| URL type | Scheme | Example |
|----------|--------|---------|
| External | `https://` only | `https://api.example.com/v1` |
| ngrok internal | `http://` or `https://` | `https://my-service.internal` |

HTTP is only allowed for ngrok `.internal` endpoints. External URLs must use HTTPS.

## Route traffic to a custom provider

1. [Add a provider key](/ai-gateway/guides/attaching-provider-keys#add-a-provider-key) if the upstream requires authentication.
2. Create an [access key configuration](/ai-gateway/guides/access-key-configurations) with:
   - **Access** allowing your custom provider ID
   - A **routing rule** with a `user` step referencing the provider key
3. Assign the configuration to an access key.
4. Call the gateway with `provider:model` in the request:

```json
{
  "model": "my-ollama:llama3.2",
  "messages": [{"role": "user", "content": "Hello"}]
}
```

## Self-hosted on a local network

To reach models on your machine or private network, expose the service with an ngrok internal endpoint, then use that URL as the custom provider's `baseUrl`. Reaching internal endpoints may require a ngrok platform plan—see [Credits](/ai-gateway/concepts/credits).

See the [Ollama guide](/ai-gateway/custom-providers/ollama) for a full walkthrough.
