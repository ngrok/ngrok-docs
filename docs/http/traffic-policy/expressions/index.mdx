# Overview

The Traffic Policy module enables you to filter inbound and outbound traffic with [Common Expression Language (CEL)](https://github.com/google/cel-spec) expressions. Each policy rule expression must evaluate to true in order for
the rule's actions to take effect against traffic.

- [Learn how to write CEL expressions](/docs/http/traffic-policy/expressions/writing-guide).

In addition to CEL's [built-in functions and macros](https://github.com/google/cel-spec/blob/master/doc/langdef.md#macros) we provide additional variables and macros:

- [Action Result Variables](/docs/http/traffic-policy/expressions/variables#action-result-variables)
- [Connection Variables](/traffic-policy/variables/connection#connection-variables)
- [Connection Geo Variables](/traffic-policy/variables/connection#connection-geo-variables)
- [Connection TLS Variables](/traffic-policy/variables/connection#connection-tls-variables)
- [Connection TLS Client Variables](/traffic-policy/variables/connection#connection-tls-client-variables)
- [Connection TLS Server Variables](/traffic-policy/variables/connection#connection-tls-server-variables)
- [Endpoint Variables](/traffic-policy/variables/endpoint)
- [Request Variables](/docs/http/traffic-policy/expressions/variables#request-variables)
- [Response Variables](/docs/http/traffic-policy/expressions/variables#response-variables)
- [Time Variables](/traffic-policy/variables/time)
- [List of available macros for HTTP](/traffic-policy/macros)
