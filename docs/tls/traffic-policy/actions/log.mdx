# Log

## Overview

The Log policy action allows you to leverage ngrok's [eventing system](/obs) to introduce observability into your traffic policy.

## Example

<ExampleTLS />

## Behavior

When this action is executed, the specified metadata will be collected on the [tcp_connection_closed.v0](/obs/reference/#tcp_connection_closedv0) event
under `traffic_policy.logs`.

## Reference

### Supported Directions

- Inbound

### Configuration

| Type  |
| ----- |
| `log` |

| Parameter  | &nbsp;                 | Description                                                                            |
| ---------- | ---------------------- | -------------------------------------------------------------------------------------- |
| `metadata` | Map&lt;string, any&gt; | A map of metadata that you would like to be included in the details of your log event. |
