# Log

## Overview

The Log policy action allows you to leverage ngrok's [eventing system](/obs) to introduce observability into your traffic policy.

## Behavior

When this action is executed, a [log event](/obs/reference#log) will be issued using the specified configuration.

### Configuration

| Type  |
| ----- |
| `log` |

| Parameter    | Description                                                                            |
| ------------ | -------------------------------------------------------------------------------------- |
| **metadata** | A map of metadata that you would like to be included in the details of your log event. |
