ngrok's API gateway lets you centralize traffic management for all your APIs in one placeâ€”no matter where they run.

- Route traffic to upstream services based on path, headers, or any request property.
- Authenticate requests with JWT validation, OAuth, OIDC, or API keys.
- Apply rate limiting, IP restrictions, and WAF rules to protect your APIs.
- Observe all requests from a single dashboard with [Traffic Inspector](/obs/traffic-inspection/).

## Concepts

<Columns cols={1}>
	<Card title="Cloud Endpoints" href="/universal-gateway/cloud-endpoints/" horizontal>
		Always-on, public-facing endpoints where you centrally manage traffic policy and route requests to your upstream services.
	</Card>
	<Card title="Internal Endpoints" href="/universal-gateway/internal-endpoints/" horizontal>
		Private endpoints for your upstream API services that receive traffic forwarded from Cloud Endpoints.
	</Card>
	<Card title="Traffic Policy" href="/traffic-policy/" horizontal>
		Rules you attach to endpoints to authenticate, rate limit, transform, and route traffic.
	</Card>
	<Card title="Forward Internal" href="/traffic-policy/actions/forward-internal/" horizontal>
		The action that routes traffic from your public Cloud Endpoint to internal Agent Endpoints.
	</Card>
</Columns>
