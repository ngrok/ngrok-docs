## Configuration Reference

This is the [Traffic Policy](/traffic-policy/) configuration reference for this action.

### Action Type

`terminate-tls`

### Configuration Fields

<Config>
    <ConfigField title="min_version" type="string" required={false}>
        <p>The minimum TLS version to support. Must be one of `1.0`, `1.1`, `1.2`, or `1.3`. Defaults to `1.2`.</p>
    </ConfigField>

    <ConfigField title="max_version" type="string" required={false}>
        <p>The maximum TLS version to support. Must be one of `1.0`, `1.1`, `1.2`, or `1.3`. Defaults to `1.3`.</p>
    </ConfigField>

    <ConfigField title="server_private_key" type="string" required={false} cel={true}>
        <p>The PEM-encoded private key for the server if using a custom certificate (must be specified with `server_certificate`).</p>
        <p>Supports [CEL Interpolation](/traffic-policy/concepts/cel-interpolation).</p>
    </ConfigField>

    <ConfigField title="server_certificate" type="string" required={false} cel={true}>
        <p>The PEM-encoded certificate for the server if using a custom certificate (must be specified with `server_private_key`).</p>
        <p>Supports [CEL Interpolation](/traffic-policy/concepts/cel-interpolation).</p>
    </ConfigField>

    <ConfigField title="mutual_tls_certificate_authorities" type="array of strings" required={false} cel={true}>
        <p>A list of PEM-encoded Certificate Authority certificates and/or Certificate Authority IDs that are trusted for mutual TLS authentication.</p>
        <p>Supports [CEL Interpolation](/traffic-policy/concepts/cel-interpolation).</p>
    </ConfigField>

    <ConfigField title="mutual_tls_verification_strategy" type="string" required={false}>
        <p>The strategy to use for mutual TLS verification. Must be one of `require-and-verify`, `require-any`, or `request`. Defaults to `require-and-verify`.</p>
    </ConfigField>

</Config>

### Supported Phases

- `on_tcp_connect`

### Supported Schemes

- `https`
- `tls`
