h2: Frequently asked questions

What is a device gateway?
A device gateway is a way to securely connect to devices deployed in remote networks, like sensors, controllers, kiosks, gateways, embedded systems. ngrok acts as the gateway by giving each device a secure, addressable endpoint without requiring firewall changes or VPNs on the network where the device lives.

What devices and protocols does ngrok support?
Any device that runs the agent or SDK, which includes ARM boards, Raspberry Pis, embedded Linux, Windows servers, industrial controllers. HTTP, TCP, TLS, and SSH natively. UDP and proprietary protocols like Modbus work through TCP tunnels. Your technicians keep their existing RDP clients, SSH terminals, and thick-client apps.

How is this different from a VPN?
VPNs require the network owner to configure them, grant broad network access, and are operationally painful to maintain across heterogeneous customer networks. ngrok's agent connects outbound on port 443, which means you don’t need to reconfigure your remote networks, scopes access to specific services, not entire devices.

Can my cloud services access device APIs through ngrok?
Yes. Each device gets an addressable endpoint that your cloud services can call like any other API to query sensor data, push configuration, trigger actions, and so on. You can keep these endpoints always-on or create them on demand.

How do I manage credentials across thousands of devices?
ngrok provides API-driven credential management. Create per-device or per-customer auth tokens from a single master account, scope them with ACL rules, and rotate or revoke them at any time without physical access to the device.

Can I embed ngrok into my own product?
Yep. The ngrok agent is available as an SDK (Go, Python, Rust, Java) that you compile into your own device software. You can white-label all URLs and the agent connect address so your customers never see ngrok's brand.

How do I manage credentials and policy across thousands of devices?
Create per-device or per-customer auth tokens from one account, scope them with ACLs, rotate or revoke any time without having to physically access the device. Traffic Policy is configured in ngrok's cloud and enforced at the edge, so when you update auth rules or rate limits, changes apply across your fleet immediately without firmware pushes.

Where does my device traffic travel?
Traffic flows through the ngrok cloud, which operates points of presence globally. With TLS endpoints, ngrok routes encrypted bytes without terminating the connection. You can restrict traffic to specific regions for data residency.

What is the pricing for device connectivity?
ngrok offers usage-based pricing measured in active endpoint hours, which means you only pay when devices are actively sending or receiving traffic. This is designed for device fleets where you have thousands of endpoints are deployed but only request or access them infrequently. See the pricing page for more details.
